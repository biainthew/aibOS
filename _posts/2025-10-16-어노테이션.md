### 사용 예시

1. 해당 요소의 사용 목적이나 의미를 명시하기 위해 사용된다

    {% raw %}
```java
    @Override
    public void myMethod() {
        // 부모 클래스의 myMethod를 오버라이드
    }
    ```
{% endraw %}

2. Spring, Hibernate 와 같은 프레임워크에서 어노테이션을 사용하여 설정을 단순화하고 코드의 기능을 향상시킨다

    {% raw %}
```java
    @Entity
    public class User {
        @Id
        private Long id;
    
        @Column(name = "username")
        private String username;
    }
    ```
{% endraw %}

3. 커스텀 어노테이션

    {% raw %}
```java
    @Retention(RetentionPolicy.RUNTIME)
    @Target(ElementType.METHOD)
    public @interface MyCustomAnnotation {
        String value();
    }
    ```
{% endraw %}


### 종류


**@Entity :**


JPA(Java Persistence API) 에서 사용되며 해당 클래스가 데이터베이스의 엔티티임을 나타낸다


**@Override :** 


메서드가 부모 클래스의 메서드를 오버라이드 하고 있음을 나타낸다


**@Deprecated :** 


더 이상 사용되지 않는 메서드, 클래스, 또는 필드


**@Table :**


JPA 에서 사용되며 해당 엔티티가 매핑될 테이블의 이름을 정의한다


**@Column :**


JPA 에서 사용되며 클래스의 필드가 데이터베이스 테이블의 열과 매핑됨을 나타낸다


**@Autowired :** 


Spring Framework에서 의존성 주입을 나타낸다 해당 필드나 메서드에 Spring이 자동으로 의존성을 주입


**@RequestMapping :**


Spring MVC 에서 HTTP 요청을 처리하는 메서드를 지정 URL과 HTTP 메서드(GET, POST 등)를 매핑할 수 있다


**@Configuration :** 


Spring Framework 에서 사용되며 해당 클래스가 하나 이상의 @Bean 메서드를 포함하는 설정 클래스를 나타낸다 이 클래스는 Spring IoC 컨테이너에 의해 관리되며 어플리케이션의 구성 정보를 제공한다


**@Return :** 


메서드의 반환값을 설명하는데 사용


**@Param :** 


Spring Framework 에서 사용하며 메서드 매개변수에 대한 추가 정보를 제공한다 @RequestParam 어노테이션과 함께 사용되어 HTTP 요청의 파라미터를 메서드의 매개변수에 바인딩


{% raw %}
```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class MyController {
    
    @GetMapping("/greet")
    public String greet(@RequestParam(name = "name", defaultValue = "World") String name) {
        return "Hello, " + name + "!";
    }
}
```
{% endraw %}


**@Transactional :**


트랜잭션 관리


**@RequiredArgsConstructor :** 


lombok으로 스프링에서 의존성 주입하는 방법 중 생성자 주입을 임의의 코드 없이 자동으로 설정해준다


초기화되지 않은 final 필드나 NonNull 이 붙은 필드에 대해 생성자를 생성해준다


새로운 필드를 추가할 때 다시 생성자를 만들어서 관리해야 하는 번거로움을 없애준다(@Autowired를 사용하지 않고 의존성 주입)


{% raw %}
```java
@RestController
@RequiredArgsConstructor
@RequestMapping("/example")
public class RequiredArgsConstructorControllerExample {

  private final FirstService firstService;
  private final SecondService secondService;
  private final ThirdService thirdService;
  
  ...
}
```
{% endraw %}
