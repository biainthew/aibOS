### 도커란 ?


리눅스 컨테이너 기술을 기반으로 한 OS 레벨 가상화 도구로 어플리케이션을 실행하는 데 필요한 모든 환경을 하나로 모아 어디서든 동일하게 실행할 수 있도록 돕는다


**비유하자면 밀키트 같은 것**

- **구성 요소** : OS, 라이브러리, 프로그램, SDK, 실행 코드가 모두 포함됨
- **이미지 (밀키트)** : 실행 가능한 패키지 상태
- **컨테이너 (조리된 요리)** : 이미지를 실행하여 실제로 돌아가는 가상 컴퓨터

VM(가상 머신) vs Docker(컨테이너)


| **항목** | **VM (Virtual Machine)** | **Docker (Container)** |
| ------ | ------------------------ | ---------------------- |
| 가상화 수준 | 하드웨어 레벨 (Guest OS 포함)    | OS 레벨 (커널 공유)          |
| 속도     | 부팅 및 실행이 무거움             | 매우 빠르고 가벼움             |
| 자원 사용  | 설정된 자원을 미리 점유            | 필요한 만큼만 동적으로 사용        |


### 도커를 사용하는 이유

1. **환경 일관성 및 용이성**

    개발 환경에서는 잘 되는데 서버에서는 안 되는 문제를 방지한다. Dockerfile 을 통해 생성된 이미지는 어떤 환경에서도 잘 동작한다.

2. **MSA (Microservices Architecture) 도입**

    과거에는 하나의 거대한 프로그램에 모든 기능을 넣었으나 최근에는 기능을 잘게 쪼개어 여러 프로그램으로 나누는 MSA 가 주류인데 도커는 이 수많은 작은 프로그램들을 관리하고 배포하기에 가장 적합한 기술이다.

3. **CI/CD 자동화**

    코드만 올리면 자동으로 도커 이미지를 빌드하고 서버에 배포하는 자동화 파이프라인 구축이 쉬워진다.

4. **데이터 휘발성 및 관리**

    컨테이너는 삭제되면 내부 데이터가 모두 삭제되는 휘발성을 가진다. 이는 시스템을 항상 깨끗한 상태로 유지할 수 있게 해주며 필요한 데이터는 Volume 설정을 통해 별도로 보존한다.


### 추가 개념들


인프라 구성 및 빌드

- Dockerfile : 도커 이미지를 만들기 위한 설계도로 어떤 OS 를 쓸 지 어떤 파일을 복사할 지 등의 명령어를 기록
- nginx, server, DB 구성 : 웹서버, 어플리케이션 서버, 데이터베이스를 각각의 컨테이너로 분리하여 유기적으로 연결하는 표준 구성
- Multi-stage build : 빌드 과정에서 필요한 도구들과 실제 실행에 필요한 결과물을 분리하여 최종 이미지 용량을 최소화하는 기법

컨테이너 통신 및 저장

- **Volume** : 컨테이너가 삭제되어도 데이터를 유지하기 위해 호스트 OS 의 저장공간과 연결하는 기능 (DB 데이터 저장 등에 필수)
- **Network** : 여러 개의 컨테이너가 서로 통신할 수 있도록 가상의 네트워크를 설정하는 기능

이미지 관리 및 운영

- **Push/Pull** : 생성한 이미지를 원격 저장소에 올리거나 저장소에서 이미지를 내려받는 동작
- **Docker Compose** : 여러 개의 컨테이너를 하나의 yaml 파일로 정의하고 한번에 실행/관리할 수 있게 해주는 툴
- **Watch** : 파일 변경을 감지하여 컨테이너 상태를 자동으로 업데이트하는 기능
- **Healthcheck** : 컨테이너 내부의 프로세스가 정상적으로 동작 중인지 주기적으로 체크하는 설정
- **Replication** : 고가용성을 위해 동일한 컨테이너를 여러 개 복제하여 실행하는 방식

확장 및 관리

- **Orchestration** : 수많은 컨테이너의 배포, 확장, 네트워크, 상태 관리 등을 자동화 하는 기술 (대표적으로 쿠버네티스가 있음)
