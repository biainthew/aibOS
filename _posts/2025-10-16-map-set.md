### Set (집합)


: 중복을 허용하지 않는 순서 없는 값들의 집합

1. 중복된 값은 자동으로 제거됨
2. 값만 저장하고 키는 없음 (객체와 다름)
3. 순서가 보장되지 않음
4. `add()`로 값 추가, `has()`로 존재 여부 확인

**예시 : 배열에서 중복제거**


```javascript
const numbers = [1, 2, 2, 3, 4, 4, 5];
const uniqueNumbers = [...new Set(numbers)]; 
console.log(uniqueNumbers); // [1, 2, 3, 4, 5]
```


**주요 메서드**


```javascript
const mySet = new Set();

mySet.add(1); // 값 추가
mySet.add(2);
mySet.add(2); // 중복이므로 무시됨
console.log(mySet.has(2)); // true (값이 존재하는지 확인)
mySet.delete(1); // 값 삭제
console.log(mySet.size); // 1 (현재 요소 개수)
mySet.clear(); // 전체 삭제
```


### Map (키 - 값)


: 키와 값을 저장하는 객체와 유사한 구조

1. `Object` 와 달리 모든 타입의 키 (객체, 배열 등) 사용 가능
2. 키 순서가 유지됨
3. `get()` 으로 값 조회 가능
4. `has()` 로 특정 키 존재 여부 확인

**예시 : 기본적인 Map 사용**


```javascript
const myMap = new Map();

myMap.set("name", "Alice"); // 키-값 추가
myMap.set(1, "One"); // 숫자 키도 가능
myMap.set({ key: "value" }, "An object key"); // 객체 키도 가능

console.log(myMap.get("name")); // "Alice"
console.log(myMap.has(1)); // true
console.log(myMap.size); // 3 (요소 개수)
myMap.delete("name"); // 특정 키 삭제
myMap.clear(); // 전체 삭제
```


**예시 : Map 을 이용한 중복제거**


```javascript
const users = [
    { id: 1, name: "Alice" },
    { id: 2, name: "Bob" },
    { id: 1, name: "Alice" }, // 중복된 id
];

const uniqueUsers = Array.from(new Map(users.map(user => [user.id, user])).values());

console.log(uniqueUsers);
// [
//   { id: 1, name: "Alice" },
//   { id: 2, name: "Bob" }
// ]
```


### 결론


`Set` 은 중복 자동으로 제거, 순서 보장 x,  값만 저장
