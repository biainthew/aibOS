### DTO에서 VO로 변환하는 이유

1. **역할 분리**

    DTO는 클라이언트와 서버 간의 데이터 전송을 위한 객체이다 즉 API 호출 시 데이터를 주고받기 위해 사용된다 반면 VO는 비즈니스 로직에서 사용하는 객체로 주로 데이터베이스와 상호작용 할 때 사용된다 각 계층에서 객체의 역할을 명확히 분리하면 어플리케이션의 유지보수가 쉬워진다

2. **데이터 유효성**

    DTO는 외부에서 들어오는 데이터이기 때문에 데이터의 구조나 타입이 서버에서 기대하는 형식과 다를 수 있다 VO는 서버 내부에서 사용하기 때문에 더욱 엄격한 데이터 구조와 유효성 검사가 가능하다 DTO 에서 VO로 변환하는 과정에서 데이터 유효성 검사를 할 수 있다

3. **보안**

    클라이언트로부터 받은 DTO는 신뢰할 수 없는 데이터 일 수 있다 VO는 서버 내부의 안전한 데이터 구조로 클라이언트에서 온 데이터를 VO로 변환할 때 불필요한 정보나 민감한 정보를 걸러낼 수 있다

4. **캡슐화**

    DTO는 단순한 데이터 전달 역할을 하지만 VO는 엔티티의 데이터와 관련된 로직을 포함할 수 있다 이를 통해 비즈니스 로직을 더 명확하게 분리하고 유지보수성을 높일 수 있다


이렇게 DTO와 VO를 분리하면 어플리케이션 아키텍쳐가 명확해지고 확장성과 유지보수성이 높아진다
