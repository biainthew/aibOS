DB 부하를 줄이기 위해 게시글 조회 시 DB에 바로 업데이트 하지 않고 Redis 에 조회수를 쌓아두다가 일정 주기로 DB에 반영하는 방식으로 구현함


### RedisConfig 


{% raw %}
```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.connection.RedisConnectionFactory;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.serializer.StringRedisSerializer;

@Configuration
public class RedisConfig {

    @Bean
    public RedisTemplate<String, String> redisTemplate(RedisConnectionFactory connectionFactory) {
        RedisTemplate<String, String> template = new RedisTemplate<>();
        template.setConnectionFactory(connectionFactory);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(new StringRedisSerializer());
        return template;
    }
}
```
{% endraw %}

- `RedisTemplate<String, String>`  : Redis 의 key 와 value 를 모두 String 타입으로 선언
- `setConnectionFactory` : Redis 서버와 연결을 담당하는 Factory 를 설정
- `setKeySerializer` : Redis 에 저장될 key 를 직렬화 하는 방식 설정
- `setValueSerializer` : Redis 에 저장될 value 를 직렬화 하는 방식 설정

> 💡 **`StringRedisSerializer`** **를 사용하는 이유 ?**  
> Redis 는 내부적으로 데이터를 Binary 형태로 저장함  
>   
> 그래서 직렬화 설정을 하지 않으면 Java 기본 직렬화가 적용되어 Redis 에 데이터가 들어갈 때 \xac\xed\x00\x05 같은 것들이 붙어서 저장됨


### ViewCountService


{% raw %}
```java
import aib.noticeboard.domain.enums.PostStatus;
import aib.noticeboard.repository.PostRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.Set;

@Service
@RequiredArgsConstructor
public class ViewCountService {

    private final RedisTemplate<String, String> redisTemplate;
    private final PostRepository postRepository;

    private static final String VIEW_COUNT_KEY = "post:viewcount";

    public void increaseViewCount(Long postId) {
        String key = VIEW_COUNT_KEY + postId;
        redisTemplate.opsForValue().increment(key);
    }

    public int getViewCount(Long postId) {
        String key = VIEW_COUNT_KEY + postId;
        String value = redisTemplate.opsForValue().get(key);
        return value == null ? 0 : Integer.parseInt(value);
    }

    @Scheduled(fixedDelay = 60000) // 1분마다 DB에 반영
    @Transactional
    public void syncViewCountToDB() {
        Set<String> keys = redisTemplate.keys(VIEW_COUNT_KEY + "*");
        if (keys == null || keys.isEmpty()) return;

        for (String key : keys) {
            String value = redisTemplate.opsForValue().get(key);
            if (value == null) continue;

            Long postId = Long.parseLong(key.replace(VIEW_COUNT_KEY, ""));
            int count = Integer.parseInt(value);

            postRepository.findByIdAndStatus(postId, PostStatus.ACTIVE).ifPresent(post -> {
                post.syncViewCount(count);
                redisTemplate.delete(key);
            });
        }
    }
}
```
{% endraw %}

- `increaseViewCount` : Redis 메모리 상에서 조회수 증가
- `getViewCount` : Redis 에 저장된 해당 게시글의 실시간 조회수를 가져옴
- `syncViewCountToDB` : 1분마다 Redis 에 저장된 데이터를 DB에 반영하는 스케줄러

> 💡 **`opsForValue()`** **란 ?**  
> `RedisTemplate` 에서 제공하는 메서드로 Redis 의 5가지 데이터 구조 중 가장 기본인 String(Key - Value) 형태의 데이터를 다루기 위한 인터페이스  
>   
> **핵심 기능** : 특정 키에 값을 저장(`save`), 조회(`get`), 숫자형 데이터의 원자적 증가(`increment`) 연산 수행  
>   
> **조회수 로직에서 역할** : 게시글 ID 를 키로 하여 단순한 정수형 조회수 데이터를 빠르고 효율적으로 관리하기 위해 사용
