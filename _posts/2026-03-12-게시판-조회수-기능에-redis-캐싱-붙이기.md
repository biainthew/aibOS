DB 부하를 줄이기 위해 게시글 조회 시 DB에 바로 업데이트 하지 않고 Redis 에 조회수를 쌓아두다가 일정 주기로 DB에 반영하는 방식


### RedisConfig 


{% raw %}
```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.connection.RedisConnectionFactory;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.serializer.StringRedisSerializer;

@Configuration
public class RedisConfig {

    @Bean
    public RedisTemplate<String, String> redisTemplate(RedisConnectionFactory connectionFactory) {
        RedisTemplate<String, String> template = new RedisTemplate<>();
        template.setConnectionFactory(connectionFactory);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(new StringRedisSerializer());
        return template;
    }
}
```
{% endraw %}

- `RedisTemplate<String, String>`  : Redis 의 key 와 value 를 모두 String 타입으로 선언
- `setConnectionFactory` : Redis 서버와 연결을 담당하는 Factory 를 설정
- `setKeySerializer` : Redis 에 저장될 key 를 직렬화 하는 방식 설정
- `setValueSerializer` : Redis 에 저장될 value 를 직렬화 하는 방식 설정

> 💡 **`StringRedisSerializer`** **를 사용하는 이유 ?**  
> Redis 는 내부적으로 데이터를 Binary 형태로 저장함  
>   
> 그래서 직렬화 설정을 하지 않으면 Java 기본 직렬화가 적용되어 Redis 에 데이터가 들어갈 때 \xac\xed\x00\x05 같은 것들이 붙어서 저장됨
