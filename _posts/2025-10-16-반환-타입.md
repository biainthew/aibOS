메서드의 반환 타입은 메서드가 수행한 결과를 호출한 곳으로 돌려주는 데이터의 종류를 의미한다 모든 메서드는 반드시 반환 타입을 명시해야 한다


### 종류

1. 기본 데이터 타입
    1. 메서드가 `int`, `double`, `boolean` 등 기본 타입을 반환하는 경우 해당 타입의 값을 메서드가 반환한다
    2. 예) `public int add(int a, int b) {return a + b;}`
2. 참조 타입
    1. 메서드가 배열, 클래스, 인터페이스, 컬렉션과 같은 참조 타입을 반환할 수 있다
    2. 예) `public String getName() {return “Alice”;}`
3. `void` (반환값 없음)
    1. 메서드가 어떤 값도 반환하지 않을 때 사용한다 메서드 내 작업이 종료되면 바로 제어가 호출한 곳으로 돌아간다
    2. 주로 데이터를 변경하거나 출력하는 등의 작업에 사용된다

### 참조 타입 1. Class


클래스는 자바에서 참조 타입의 대표적인 예이다 클래스는 객체를 생성하기 위한 설계도로 이를 통해 다양한 속성(필드)과 기능(메서드)을 정의할 수 있다


클래스 타입의 변수는 해당 클래스의 인스턴스를 참조한다


{% raw %}
```java
public class Person {
    String name;
    int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// Person 타입의 참조 변수
Person person = new Person("Alice", 25);
```
{% endraw %}


### 참조 타입 2. 배열(Array)


배열은 동일한 타입의 여러 요소를 저장하는 자료 구조로 배열 역시 참조 타입으로 분류된다


배열은 고정된 크기로 선언되며 생성 후 크기를 변경할 수 없다


배열은 특정 타입을 지정하여 원시 타입 또는 참조 타입을 요소로 가질 수 있다


{% raw %}
```java
// 정수형 배열 (기본 타입의 배열)
int[] numbers = {1, 2, 3, 4, 5};

// 클래스형 배열 (참조 타입의 배열)
Person[] people = new Person[3];
people[0] = new Person("Bob", 30);
people[1] = new Person("Alice", 25);
```
{% endraw %}


### 참조 타입 3. 인터페이스(Interface)


인터페이스는 클래스가 특정 메서드를 구현하도록 강제하는 계약을 정의하는 참조 타입이다


인터페이스는 다형성을 제공하며 여러 클래스에서 공통으로 구현할 수 있다


인터페이스 타입의 변수는 그 인터페이스를 구현한 클래스의 인스턴스를 참조할 수 있다


{% raw %}
```java
public interface Animal {
    void sound();
}

public class Dog implements Animal {
    public void sound() {
        System.out.println("Woof!");
    }
}

// 인터페이스 타입의 참조 변수
Animal animal = new Dog();
```
{% endraw %}


### 참조 타입 4. 열거형(Enumeration, `enum`)


열거형은 특정한 상수 집합을 정의하는 참조 타입이다


`enum`은 한정된 값들의 집합을 정의하고 싶을 때 사용한다


예를 들어 요일이나 방향 등을 `enum`으로 정의할 수 있다


{% raw %}
```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

// enum 타입의 참조 변수
Day today = Day.MONDAY;
```
{% endraw %}


### 참조 타입 5. 자바 래퍼 클래스(Wrapper Classes)


자바는 각 기본 데이터 타입(`int` `char` `boolean` 등) 에 대한 래퍼 클래스를 제공한다


래퍼 클래스는 기본 타입을 객체로 다룰 수 있도록 해준다 예를 들어 `Integer`, `Double`, `Boolean` 등은 각각 `int`, `double`, `boolean` 의 래퍼 클래스이다


{% raw %}
```java
Integer number = 5;  // int를 Integer로 자동 변환
Boolean flag = true; // boolean을 Boolean으로 자동 변환
```
{% endraw %}


### 참조 타입 6. 문자열(String)


String 은 자바에서 특수하게 처리되는 참조 타입이다 불변 객체로 설계되어 있어 문자열 값이 변경되지 않는다


String 객체는 참조 타입이므로 문자열을 비교할 때는 == 대신 .equals() 메서드를 사용해야 한다


{% raw %}
```java
String greeting = "Hello, world!";
```
{% endraw %}


### 참조 타입 7. 제네릭(Generic) 타입


제네릭 타입은 클래스나 메서드가 특정 타입에 종속되지 않고 다양한 타입을 지원하도록 설계할 때 사용된다


예를 들어 `List<String>`, `List<Integer>` 와 같이 제네릭을 사용하면 타입 안전성을 높일 수 있다


{% raw %}
```java
import java.util.ArrayList;
import java.util.List;

List<String> names = new ArrayList<>();
names.add("Alice");
names.add("Bob");
```
{% endraw %}


### 참조 타입 8. 컬렉션 프레임워크 (Collections)


자바의 컬렉션 프레임워크는 데이터 구조를 위한 다양한 클래스와 인터페이스(예 : List, Set, Map 등)를 제공한다


컬렉션 프레임워크는 데이터의 저장, 탐색, 수정 등을 효율적으로 처리할 수 있게 도와준다


{% raw %}
```java
import java.util.HashMap;
import java.util.Map;

Map<String, Integer> scores = new HashMap<>();
scores.put("Alice", 90);
scores.put("Bob", 85);
```
{% endraw %}
