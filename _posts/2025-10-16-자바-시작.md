### 절차적 프로그래밍


순차적으로 처리


프로그램 전체가 긴밀히 연결되도록 만드는 프로그래밍 기법


절차 = 함수


반복되는 동작을 함수 및 프로시저 형태로 모듈화하여 사용하는 방식


**장점**

1. 컴퓨터의 처리 구조와 유사해 실행속도가 빠르다
2. 각 프로그램의 흐름을 쉽게 추적할 수 있다

**단점**

1. 각 코드가 매우 유기성이 높아 유지보수가 어렵다
2. 실행 순서가 정해져 있으므로 코드의 순서가 바뀌면 동일한 결과를 보장하기 어렵다

**예시**


은행에서 계좌 관리 프로그램을 개발하려고 한다면


계좌라는 자료형과 계좌에 대한 함수를 구현해야 한다


이 두가지는 절차적 프로그래밍에서 따로 구분된다


이러한 계좌와 계좌에 대한 함수를 묶기 위해 객체 지향 프로그래밍이 등장했다


### 객체


프로그램에서 사용되는 데이터 또는 식별자에 의해 참조되는 공간


값을 저장할 변수와 수행할 메서드를 서로 연관된 것들끼리 묶어 만든다


객체 내부에 자료형 필드와 함수가 함께 존재한다 → 코드가 직관적으로 보이고 가독성 증가


### 객체지향 프로그래밍


**장점**

1. 모듈화, 캡슐화로 인해 유지보수에 용이하다
2. 객체 자체가 하나의 프로그램이기에 재사용에 용이하다

**단점**

1. 대부분 속도가 상대적으로 느리고 메모리를 많이 사용한다
2. 코드를 설계하고 작성하는데 많은 시간이 소요된다

**예시**


java, Python, C++, C#, Kotlin


**특징**

1. 추상화

    클래스를 정의할 때 불필요한 부분들은 생략하고 객체의 속성 중 중요한 것에만 중점을 두어 개략화 하는 것


    새로운 객체를 만들 때 클래스의 고유 기능만 새로 만들어주고 공통된 기능은 상속 이용 가능


    ex) 삼성폰과 아이폰이라는 객체가 있다면 공통 특징인 휴대폰으로 묶어 이름을 붙이는 것
    공통된 기능을 휴대폰에서 미리 구현을 하고 상속을 통해 삼성폰과 아이폰에 이어주면
    각자의 고유기능을 집중적으로 개발할 수 있고
    공통 기능을 다시 정의할 필요가 없기 때문에 코드 중복 감소 재사용 용이

2. 상속
1. 다형성

    같은 자료형에 여러 가지 타입의 데이터를 대입하여 다양한 결과를 얻어낼 수 있는 성질


    객체도 상황에 따라 여러가지 형태를 가질 수 있다


    **메서드 오버라이딩**


    부모 클래스의 메서드를 자식 클래스에서 재정의해서 사용하는 것
    자식 클래스는 독자적인 기능을 추가할 수 있다


    **메서드 오버로딩**


    같은 이름의 메서드를 다양한 매개변수 타입과 개수로 오버로딩하여 사용하는 것을 말하며
    메서드 이름은 동일하게 유지하면서도 다양한 상황에서 유연하게 대응할 수 있는 방법을 제공

### 클래스


프로그램에서 사용될 객체의 틀을 정의


객체를 생성하기 위한 설계도 역할


여러 멤버 변수와 메서드를 포함하며 그것들을 적절히 구성하여 객체의 동작을 정의


**왜 쓰나요?**

1. 코드 재사용 가능
2. 유지보수성 향상
3. 코드 안정성 증가 : 각 클래스는 자체적으로 캡슐화 되어있어 다른 클래스와 독립적으로 작동함

**언제 쓰나요?**

1. 객체 생성할 때
2. 코드 구조 정리할 때
3. 코드 재사용성 높일 때

### 패키지(Package)


클래스의 묶음


클래스를 용도별이나 기능별로 그룹화 한 것


물리적으로 하나의 디렉토리이다


클래스를 유일하게 만들어주는 식별자 역할


### 접근 제어자


**public**


해당 멤버가 모든 클래스에서 접근 가능


**private**


해당 멤버가 해당 클래스 내부에서만 접근 가능


이를 통해 데이터 캡슐화를 구현하고 외부에서 직접 데이터를 수정하는 것을 막을 수 있다

### 참조 타입(Reference Type)


대표적으로 자바에서 제공하는 String
