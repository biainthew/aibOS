### 모델링


단순화, 추상화, 명확화


중복, 비유연성, 비일관성


엔티티, 속성, 관계


개념적, 논리적, 물리적


### 엔티티


엔티티 속성 식별자 인스턴스


유일한 식별자 (이름 놉), 해당 업무에 관련, 인스턴스 두 개 이상, 속성 두 개 이상, 안쓰는 엔티티는 제거 고려, 관계 한 개 이상


유형(사원, 물품), 개념(조직, 보험상품), 사건(주문, 청구, 미납)


기본(원래 존재 : 사원, 부서, 고객), 중심(파생 : 계약, 청구), 행위(두 개 이상의 부모로부터 발생 : 주문, 사원변경이력, 이력)


현업사용 용어, 약어 사용 자제, 단수 명사, 유일성, 생성 의미대로 이름 부여


엔터티 사각형


주 식별자 #


### 속성


주 식별자에 함수적 종속성을 가져야 한다


기본속성, 설계속성, 파생속성(계산된 값)


PK, FK, 일반


단일, 복합(주소), 다중값(상품 리스트)


해당 업무에서 사용, 서술식 안됨, 약어 안됨, 유일


### 도메인


속성에 대한 데이터 타입, 크기, 제약사항지정


### 관계


존재적, 행위적 → ERD 에서는 구분하지 않는다


관계명, 차수, 선택성


페어링 : 인스턴스가 개별적으로 관계를 가지는것, 연결고리


### 식별자


유일한 식별자 존재해야함, 논리모델링의 식별자는 물리모델링의 키


주 식별자 : 유일성, 최소성, 불변성, 존재성


대표성 여부 : 주식별자 보조식별자(주민번호)


생성 여부 : 내부식별자 외부식별자(외래키 우편번호)


속성 수 : 단일식별자 복합식별자


대체 여부 : 본질식별자 인조식별자


해당 업무에서 자주 이용, 이름 놉, 속성 수 최대한 적게


강한 개체, 약한 개체


### 식별관계


기본키를 다른 엔티티가 기본키의 하나로 공유


실선으로 표시


사원과 교육이력 엔티티에서 양쪽 모두 기본키중 일부가 사원번호임

### 비식별관계


강한 개체의 기본키를 다른 엔티티의 기본키가 아닌 일반 속성으로 관계를 가지는 것


점선으로 표시


**기본키로 가지냐 일반속성으로 가지냐의 차이**


### 정규화


엔티티 분해하는 과정


최소한의 데이터만을 하나의 엔티티에 넣음, 중복 제거, 독립성 확보, 이상현상 줄이기 위한, 논리 데이터 모델링 수행 시점


### 이상현상


정규화를 하지 않아 발생하는 현상


삽입이상, 갱신이상, 삭제이상


### 제1정규화


테이블의 컬럼이 원자성(한 속성이 하나의 값을 갖는 특성)을 갖도록 테이블을 분해하는 단계


하나의 행과 컬럼의 값이 반드시 한 값만 입력되도록 생을 분리하는 단계


### 제2정규화


완전함수종속 : 기본키를 구성하는 모든 컬럼의 값이 다른 컬럼을 결정짓는 단계


### 제3정규화


이행적 종속


### 관계


논리적인 연관성


부모의 식별자를 자식에 상속하고 조인키로 연결


존재적 관계, 행위적 관계
