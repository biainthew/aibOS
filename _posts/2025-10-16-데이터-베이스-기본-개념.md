### 테이블(Table)


**정의**


데이터베이스에서 데이터를 저장하는 기본 단위이다


테이블은 행과 열로 구성되어있으며 각 행은 레코드를 나타내고 열은 특정 속성(attribute)를 나타낸다


**예시**


고객 정보를 저장하는 customers 테이블은 id, email, name 등의 열을 가질 수 있으며 각 행은 한 명의 고객에 대한 정보를 나타낸다


### 뷰(View)


**정의**


하나 이상의 테이블에서 가져온 데이터를 기반으로 생성된 가상 테이블이다


실제 데이터를 저장하지 않으며 데이터 조회를 단순화하거나 특정 데이터를 가공해서 보여줄 때 사용된다


**사용 목적**


복잡한 쿼리를 단순화하거나 보안 목적으로 특정 열이나 행만 표시할 때 유용하다


**예시**


고객 테이블에서 고객의 이름과 이메일만 표시하는 뷰를 생성할 수 있다


### 루틴(Routine)


**정의**


데이터베이스에서 반복적으로 실행되는 작업을 자동화하기 위한 프로시저(Procedure)나 함수(Function)이다 루틴은 프로그래밍 로직을 데이터베이스 안에서 실행할 수 있게 한다


**종류**


프로시저(Stored Procedure) : 미리 저장된 SQL 쿼리 세트로 특정 작업을 수행하기 위해 호출된다


함수(Function) : 값을 반환하는 루틴으로 SQL 쿼리 내에서 호출될 수 있다


**예시**


고객의 총 구매액을 계산하는 함수를 만들 수 있다


### 시퀀스(Sequence)


**정의**


고유한 숫자 값을 자동으로 생성하는 객체이다 주로 기본키(Primary Key)로 사용할 고유 값을 생성하는 데 사용된다


**사용 목적**


레코드가 추가될 때마다 고유한 숫자(예 : 1, 2, 3, …)를 자동으로 부여할 때 사용한다


**예시**


orders 테이블에 새로운 주문이 추가될 때마다 자동으로 주문 ID를 생성하는 시퀀스를 만들 수 있다
