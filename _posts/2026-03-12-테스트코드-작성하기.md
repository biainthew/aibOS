`PostService`의 게시글 작성 기능을 단위 테스트한 코드로 Mockito를 사용해 외부 의존성을 모킹하고 실제 DB 없이 서비스 로직만 검증함


### 사용 기술


| 항목        | 내용      |
| --------- | ------- |
| 테스트 프레임워크 | JUnit 5 |
| 모킹 라이브러리  | Mockito |
| 검증 라이브러리  | AssertJ |


### 기본 세팅


**@ExtendWith(MockitoExtension.class)**

- JUnit 5에서 Mockito를 사용하기 위한 확장 설정
- 이 어노테이션이 있어야 `@Mock`, `@InjectMocks`가 동작함

**@InjectMocks**


{% raw %}
```java
@InjectMocks
private PostService postService;
```
{% endraw %}

- 테스트 대상 클래스
- Mockito가 `@Mock`으로 선언된 객체들을 자동으로 주입해준다.

**@Mock**


{% raw %}
```java
@Mock
private PostRepository postRepository;

@Mock
private MemberRepository memberRepository;

@Mock
private ViewCountService viewCountService;
```
{% endraw %}

- 실제 구현체 대신 가짜 객체를 사용
- `postRepository.save()`를 호출해도 실제 DB에는 아무것도 저장되지 않음

**@BeforeEach**


{% raw %}
```java
@BeforeEach
void setUp() {
    member = Member.builder()
            .email("lba0507@gmail.com")
            .password("bia12345")
            .nickname("bia12345")
            .role(MemberRole.ROLE_USER)
            .build();

    post = Post.builder()
            .member(member)
            .title("테스트 게시글 제목")
            .content("테스트 게시글 내용")
            .status(PostStatus.ACTIVE)
            .build();
}
```
{% endraw %}

- 각 테스트 메서드 실행 전에 공통으로 쓸 `member`와 `post` 객체를 초기화
- 테스트마다 매번 새로 만들어지므로 테스트 간 데이터가 오염되지 않음

### 테스트 케이스


**게시글 작성 성공**


{% raw %}
```java
@Test
@DisplayName("게시글 작성 성공")
void create_success() {
    // given
    PostRequestDto.Create request = new PostRequestDto.Create();
    given(memberRepository.findByEmail(member.getEmail())).willReturn(Optional.of(member));
    given(postRepository.save(any(Post.class))).willReturn(post);

    // when
    PostResponseDto.Detail result = postService.create(member.getEmail(), request);

    // then
    assertThat(result.getTitle()).isEqualTo(post.getTitle());
    assertThat(result.getContent()).isEqualTo(post.getContent());
    verify(postRepository).save(any(Post.class));
}
```
{% endraw %}


**given / when / then 구조**


| 단계    | 설명                  |
| ----- | ------------------- |
| given | Mock 객체의 동작을 사전에 정의 |
| when  | 테스트할 메서드를 실제로 호출    |
| then  | 결과를 검증              |


**검증 포인트**

- `result.getTitle()`이 `post.getTitle()`과 같은지 확인
- `result.getContent()`이 `post.getContent()`과 같은지 확인
- `postRepository.save()`가 실제로 한 번 호출됐는지 `verify`로 확인

### 중요한 메서드 정리


**given().willReturn()**


{% raw %}
```java
given(memberRepository.findByEmail(member.getEmail())).willReturn(Optional.of(member));
```
{% endraw %}

- 특정 조건으로 메서드가 호출됐을 때 반환할 값을 지정

**any(Post.class)**


{% raw %}
```java
given(postRepository.save(any(Post.class))).willReturn(post);
```
{% endraw %}

- 인자가 `Post` 타입이기만 하면 어떤 객체든 매칭
- `save()`에 넘겨지는 `Post` 객체는 서비스 내부에서 생성되기 때문에, 정확한 인스턴스를 특정할 수 없어서 `any()`를 사용

**verify()**


{% raw %}
```java
verify(postRepository).save(any(Post.class));
```
{% endraw %}

- 해당 메서드가 실제로 호출됐는지 검증
- 반환값 검증과 별개로, 내부 동작(side effect)이 일어났는지 확인할 때 사용

### 참고

- `@DisplayName`은 테스트 이름을 한국어로 명시할 때 사용하고 테스트 결과 리포트에서 그대로 표시됨
- `BDDMockito.given()`은 `Mockito.when()`과 동작은 같지만 given/when/then 패턴에 맞게 가독성을 높인 버전

---


`PostController` 의 게시글 작성 API 를 테스트한 코드로 `MockMvc` 를 사용해 실제 HTTP 요청, 응답 흐름을 검증하고 Spring Security 설정까지 포함해서 테스트함


### 사용 기술


| 항목        | 내용                   |
| --------- | -------------------- |
| 테스트 프레임워크 | JUnit 5              |
| 모킹 라이브러리  | Mockito              |
| 웹 계층 테스트  | MockMvc              |
| 인증 처리     | Spring Security Test |


### Service 테스트와의 차이점


| 구분       | Service 테스트                           | Controller 테스트      |
| -------- | ------------------------------------- | ------------------- |
| 테스트 범위   | 서비스 로직                                | HTTP 요청/응답, 라우팅, 인증 |
| 사용 어노테이션 | `@ExtendWith(MockitoExtension.class)` | `@WebMvcTest`       |
| Mock 방식  | `@Mock`                               | `@MockitoBean`      |
| 검증 방식    | AssertJ                               | MockMvc + jsonPath  |


### 기본 세팅


**@WebMvcTest(PostController.class)**

- 웹 계층(Controller)만 띄우는 슬라이스 테스트
- `@Service`, `@Repository` 같은 빈은 로드하지 않아서 테스트가 가볍고 빠름
- `PostController`만 대상으로 지정했기 때문에 다른 컨트롤러는 로드되지 않음

**@Import(SecurityConfig.class)**

- `@WebMvcTest`는 Security 설정을 자동으로 불러오지 않음
- `SecurityConfig`를 명시적으로 import해서 실제 Security 필터 체인이 적용된 상태로 테스트

**@MockitoBean**


{% raw %}
```java
@MockitoBean
private PostService postService;

@MockitoBean
private JwtTokenProvider jwtTokenProvider;
```
{% endraw %}

- Spring 컨텍스트에 Mock 빈을 등록함
- Service 테스트의 `@Mock`과 달리, Spring이 관리하는 빈으로 등록되어 컨트롤러에 주입됨
- `JwtTokenProvider`는 Security 설정에서 의존하기 때문에 함께 모킹해야 컨텍스트 로드가 됨

**MockMvc / ObjectMapper**


{% raw %}
```java
@Autowired
private MockMvc mockMvc;

@Autowired
private ObjectMapper objectMapper;
```
{% endraw %}

- `MockMvc` : 실제 서버를 띄우지 않고 HTTP 요청을 시뮬레이션하는 객체
- `ObjectMapper` : 요청 바디를 JSON 문자열로 직렬화할 때 사용

### 테스트 케이스


**게시글 작성 성공**


{% raw %}
```java
@Test
@DisplayName("게시글 작성 성공")
@WithMockUser(username = "lba0507@gmail.com")
void create_success() throws Exception {
    // given
    PostRequestDto.Create request = new PostRequestDto.Create();
    PostResponseDto.Detail response = PostResponseDto.Detail.builder()
            .id(1L)
            .title("테스트 제목")
            .content("테스트 내용")
            .nickname("테스터")
            .viewCount(0)
            .likeCount(0)
            .createdAt(LocalDateTime.now())
            .updatedAt(LocalDateTime.now())
            .build();

    given(postService.create(any(), any())).willReturn(response);

    // when & then
    mockMvc.perform(post("/api/posts")
                    .contentType(MediaType.APPLICATION_JSON)
                    .content(objectMapper.writeValueAsString(request)))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.title").value("테스트 제목"));
}
```
{% endraw %}


**@WithMockUser**

- Spring Security가 적용된 상태에서 인증된 사용자를 흉내냄
- 이 어노테이션 없이 요청하면 Security 필터에 막혀서 401 또는 403이 떨어짐
- `username` 속성에 넣은 값이 `SecurityContext`의 `principal`로 설정됨

검증 포인트

- `status().isCreated()` : HTTP 응답 코드가 201인지 확인
- `jsonPath("$.title").value("테스트 제목")` : 응답 JSON의 `title` 필드값이 일치하는지 확인

### 중요한 메서드 정리


**mockMvc.perform()**


{% raw %}
```java
mockMvc.perform(post("/api/posts")
        .contentType(MediaType.APPLICATION_JSON)
        .content(objectMapper.writeValueAsString(request)))
```
{% endraw %}

- 지정한 HTTP 메서드와 URL로 요청을 보냄
- `contentType()` : 요청 헤더의 `Content-Type` 설정
- `content()` : 요청 바디 설정. `ObjectMapper`로 객체를 JSON 문자열로 변환해서 넣음

**jsonPath()**


{% raw %}
```java
.andExpect(jsonPath("$.title").value("테스트 제목"))
```
{% endraw %}

- 응답 JSON에서 특정 필드를 경로로 지정해서 값을 검증
- `$`는 JSON 루트를 의미하고, `$.title`은 최상위 `title` 필드를 가리킴
