### 이미지 호스팅 서비스 (Registry)


빌드한 이미지를 다른 컴퓨터나 서버로 전달하려면 이미지를 저장하고 관리해주는 **이미지 레지스트리**가 필요


### Docker Hub 이미지 업로드 및 다운로드 과정


1. 이미지 이름 변경 (docker tag)


로컬에서 생성한 이미지 이름과 Repository 의 주소 및 이름이 일치해야 업로드 가능


{% raw %}
```yaml
docker tag 기존이미지명:태그 사용자계정/레포지토리명:태그
```
{% endraw %}

- 한 이미지는 여러 개의 이름을 가질 수 있음

2. 이미지 업로드 (docker push)


태그를 맞춘 이미지를 원격 저장소로 push함 이미지 이름이 저장소 이름과 동일해야함


{% raw %}
```yaml
docker push 사용자계정/레포지토리명:태그
```
{% endraw %}


3. 이미지 다운로드 (docker pull)


서버 등 다른 환경에서 저장소에 올라온 이미지를 내려받음


{% raw %}
```yaml
docker pull 사용자계정/레포지토리명:태그
```
{% endraw %}


### 명령어 요약


| **명령어**         | **역할**                        |
| --------------- | ----------------------------- |
| **docker tag**  | 기존 이미지에 저장소 경로가 포함된 새 이름을 부여함 |
| **docker push** | 로컬 이미지를 원격 저장소에 업로드함          |
| **docker pull** | 원격 저장소에서 로컬로 이미지를 다운로드함       |


### 효율적인 레포지토리 관리 전략


웹 서버, 프론트엔드, 데이터베이스 등 각 서비스 구성 요소는 **별도의 레포지토리**를 생성하여 관리하는 것 권장

- 각 파트마다 기능 수정 및 배포 주기가 다름
- 서비스마다 사용하는 언어(Node.js, Java 등)와 라이브러리가 달라 베이스 이미지 구성이 다름
- 특정 파트만 업데이트할 때 해당 레포지토리만 관리하면 되므로 운영 효율이 높음 -> MSA 구조 최적화
