Javascript 에서 HTTP 요청을 만들 때 데이터를 폼 형식으로 전송할 수 있도록 도와주는 객체


주로 파일 업로드와 같은 multipart/form-data 형식의 요청을 처리할 때 사용됨


FormData 객체는 서버로 데이터를 전송할 때 자동으로 Content-Type 을 설정하고 다중 파트 데이터를 쉽게 관리할 수 있게 해줌


### 기본 사용법


폼 요소가 <form> 태그 안에 있을 때 그 데이터를 FormData 객체에 넣고 XMLHttpRequest 또는 fectch 로 서버에 전송할 수 있음

1. FormData 객체 생성

    ```javascript
    const formData = new FormData();
    ```

2. 폼 데이터 추가하기
FormData.append() 메서드를 사용하여 데이터를 추가할 수 있음
이 메서드는 이름과 값을 쌍으로 추가하며 값으로 파일도 포함할 수 있음

    ```javascript
    // 단일 값 추가
    formData.append('name', 'John Doe');
    
    // 파일 추가 (파일 선택을 위한 <input type="file">을 사용한다고 가정)
    const fileInput = document.querySelector('#fileInput');
    formData.append('file', fileInput.files[0]);
    ```

3. FormData 를 서버로 전송하기

    파일을 포함한 데이터를  서버에 전송할 때는 `FormData`를 `XMLHttpRequest`나 `fetch` API와 함께 사용할 수 있음


### 주요 메서드

1. `append(name, value)`
    1. `name` : 전송할 필드의 이름
    2. `value` : 전송할 값 (값은 파일, 텍스트 등 어떤 데이터든 가능, 객체는 넣을 수 없음)
2. `delete(name)`

    : 특정 이름의 필드를 `FormData`에서 제거

3. `get(name)`

    : `name` 에 해당하는 값을 반환, 파일이나 텍스트 값 모두 반환 가능

4. `getAll(name)`

    : 특정 이름에 대한 모든 값을 배열 형태로 반환, 동일한 이름으로 여러 값을 추가한 경우 유용

5. `has(name)`

    : name 에 해당하는 필드가 존재하는지 확인

6. `set(name, value)`

    : 특정 이름에 대해 값을 설정, 동일한 이름으로 여러 번 `append()` 를 사용한 경우 가장 마지막 값으로 덮어씀

7. `forEach(callback)`

    : Formdata에 있는 모든 항목에 대해 지정된 callback 함수를 실행, 이 메서드는 폼 데이터의 모든 항목을 반복하면서 처리할 수 있음
