### 연산자 우선순위는 연산 순서를 결정하는 것이 아니라 어떻게 묶을 것인지를 결정한다


```c
// 예시
int main() {
    int a = 5, b = 3, c = 2;
    int result;
    result = a++ + --b * c++ - ++a + b--;
    printf("%d, %d, %d, %d\n", a, b, c, result);
    return 0;
}
```


→ 처음에는 후위 연산자가 전위 연산자보다 연산자 우선순위가 높다고 알고 있어서 후위 연산자를 먼저 계산 함


→ 그래서 나온 식은 5 + 1 * 2 - 7 + 3


그런데 해설을 보니 왼쪽부터 증감연산자 계산을 해야한다고 해서 찾아본 내용


### 예시


`a + b * c` 를 예로 들었을 때


곱셈이 덧셈보다 우선순위가 높다 = `a + (b * c)` 로 묶인다는 의미


그러므로 `a++ + --b * c++` 를 예로 들면


`(a++) + ((--b) * (c++))` 이런식으로 연산자 우선순위는 어떤 값들이 함께 계산되는지를 결정한다


그리고 모든 증감 연산은 **왼쪽에서 오른쪽**으로 실행한다


### 결론


**우선순위** : 어떤 연산자와 어떤 피연산자가 묶이는 지를 결정


**실행 순서** : 실제로 어떤 것부터 계산하는지(보통 왼 → 오)


### 추가 - 연산자 우선순위 정리


1순위

- `()` 괄호
- `[]` 배열 첨자
- `>` 구조체 포인터 접근
- `.` 구조체 멤버 접근
- `++` `-` 후위 증감

2순위

- `++` `-` 전위 증감
- `+`  단항 부호
- `!` 논리 NOT
- `~` 비트 NOT
- 포인터 역참조
- `&` 주소 연산자
- `sizeof` 크기 연산자
- `(타입)` 형변환

3순위

- `/` `%` 곱셈, 나눗셈, 나머지

4순위

- `+`  덧셈, 뺄셈

5순위

- `<<` `>>` 비트 시프트

6순위

- `<` `<=` `>` `>=` 관계 연산자

7순위

- `==` `!=` 동등 비교

8순위

- `&` 비트 AND

9순위

- `^` 비트 XOR

10순위

- `|` 비트 OR

11순위

- `&&` 논리 AND

12순위

- `||` 논리 OR

13순위

- `? :` 삼항 연산자

14순위 (최하위)

- `=` `+=` `=` `=` `/=` 등 대입 연산자

15순위

- `,` 쉼표 연산자
