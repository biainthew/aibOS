Spring Boot 애플리케이션을 AWS EC2에 배포하고 RDS MySQL을 데이터베이스로 연결한 과정


## 인프라 구성


{% raw %}
```bash
GitHub Actions
     ↓ (push to main)
EC2 (Spring Boot + Redis)
     ↓
RDS MySQL
```
{% endraw %}


## EC2 설정


**인스턴스 스펙**


| 항목      | 설정값                     |
| ------- | ----------------------- |
| AMI     | Ubuntu Server 24.04 LTS |
| 인스턴스 유형 | t3.micro (프리티어)         |
| 스토리지    | 30GB                    |


**EC2 접속**


{% raw %}
```bash
ssh -i noticeboard-key.pem ubuntu@{EC2 퍼블릭 IP}
```
{% endraw %}


**Java 17 설치**


{% raw %}
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y openjdk-17-jdk
java -version
```
{% endraw %}


**Docker 설치**


{% raw %}
```bash
sudo apt install -y docker.io
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker ubuntu
```
{% endraw %}


**Redis 설치**


{% raw %}
```bash
sudo apt install -y redis-server
sudo systemctl start redis-server
sudo systemctl enable redis-server
redis-cli ping # PONG 출력되면 정상
```
{% endraw %}


Redis는 Spring Boot와 같은 EC2에 설치해서 `localhost:6379`로 연결함


트래픽이 늘어나면 AWS ElastiCache로 분리하는 게 맞지만 현재 규모에서는 같은 서버에 두는 것으로 충분하다고 판단했기 때문


> 💡 **Redis 를** [**localhost**](http://localhost/) **로 둬도 되는 이유**  
> [localhost](http://localhost/) 는 현재 이 서버 자신을 가리킴  
>   
> Spring Boot 가 EC2 위에서 실행되고 있고 Redis 도 같은 EC2 위에서 실행되고 있으니까 Spring Boot 입장에서 Redis 는 같은 서버 안에 있는 것  
> >> 그래서 [localhost](http://localhost/):6379 로 접근이 가능  
>   
> 만약 Redis 가 ElastiCache 등의 다른 서버에 있었다면 그 서버의 주소를 넣어야 함


## RDS 설정


| 항목       | 설정값                |
| -------- | ------------------ |
| 엔진       | MySQL 8.0          |
| 인스턴스 클래스 | db.t3.micro (프리티어) |
| 스토리지     | 20GB               |
| 퍼블릭 액세스  | 비활성화               |


RDS는 퍼블릭 액세스를 비활성화하고 EC2의 보안 그룹에서만 3306 포트 접근을 허용하는 방식으로 설정함


외부에서 직접 RDS에 접근할 수 없고 EC2를 통해서만 접근 가능한 구조


**EC2에서 RDS 접속 및 DB 생성**


RDS는 SSL 인증서가 필요해서 아래 명령어로 인증서를 먼저 다운로드한 후 접속함


{% raw %}
```bash
curl -o global-bundle.pem https://truststore.pki.rds.amazonaws.com/global/global-bundle.pem
mysql -h {RDS 엔드포인트} -P 3306 -u admin -p --ssl-mode=VERIFY_IDENTITY --ssl-ca=./global-bundle.pem
```
{% endraw %}


{% raw %}
```sql
CREATE DATABASE noticeboard DEFAULT CHARACTER SET utf8mb4;
exit;
```
{% endraw %}


## 환경변수 관리


레포지토리가 public이라 `application-prod.yml`에 RDS 엔드포인트나 비밀번호를 직접 넣을 수 없는 문제가 있었는데 GitHub Secrets에 민감 정보를 저장하고 GitHub Actions에서 환경변수로 주입하는 방식으로 해결함


{% raw %}
```yaml
# application-prod.yml
spring:
  datasource:
    url: ${DB_URL}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}

jwt:
  secret: ${JWT_SECRET}
```
{% endraw %}


GitHub Secrets에 저장한 항목


| Name         | 설명                    |
| ------------ | --------------------- |
| DB_URL       | RDS 엔드포인트 포함 JDBC URL |
| DB_USERNAME  | RDS 마스터 사용자 이름        |
| DB_PASSWORD  | RDS 마스터 비밀번호          |
| JWT_SECRET   | JWT 서명 시크릿 키          |
| EC2_HOST     | EC2 퍼블릭 IP            |
| EC2_USERNAME | ubuntu                |
| EC2_KEY      | .pem 파일 전체 내용         |


## 보안 그룹 설정


**EC2 인바운드 규칙**


| 유형         | 포트   | 소스        |
| ---------- | ---- | --------- |
| SSH        | 22   | 0.0.0.0/0 |
| HTTP       | 80   | 0.0.0.0/0 |
| HTTPS      | 443  | 0.0.0.0/0 |
| 사용자 지정 TCP | 8080 | 0.0.0.0/0 |


SSH를 `0.0.0.0/0`으로 열어둔 이유는 GitHub Actions 서버 IP가 매번 바뀌기 때문


내 IP만 허용하면 GitHub Actions에서 EC2로 접근이 안 됨


**RDS 인바운드 규칙**


| 유형           | 포트   | 소스           |
| ------------ | ---- | ------------ |
| MySQL/Aurora | 3306 | EC2 보안 그룹 ID |


RDS는 EC2 보안 그룹 ID를 소스로 지정해서 EC2에서만 접근 가능하도록 설정함
