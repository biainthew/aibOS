### 🌼 선언적 렌더링 🌼


→ 뷰의 핵심 기능 / HTML 을 확장하는 템플릿 문법을 사용하여 Javascript 상태를 기반으로 HTML 이 어떻게 보이는지 설명할 수 있음 / 상태가 변경되면 HTML 이 자동으로 업데이트 됨


→ 변경 시 업데이트를 트리거할 수 있는 상태는 **반응형**으로 간주됨 / 반응형 상태는 컴포넌트에 유지됨 / 예제 코드에서 `createApp()` 에 전달되는 객체는 컴포넌트임 / 컴포넌트에서 객체를 반환해야하는 함수 data 옵션을 사용하여 반응형 상태를 선언할 수 있음


→ 템플릿에서 이중 중괄호 문법 `{{ }}` 을 사용하여 변수 값을 동적으로 텍스트로 렌더링 가능


→ 이중 중괄호 내부의 내용은 식별자, 경로, 유효한 Javascript 표현식 사용 가능


```javascript
<script>
export default {
  data() {
    return {
      message: '안녕 Vue!',
      counter: {
        count: 0
      }
    }
  }
}
</script>

<template>
  <h1>{{ message }}</h1>
  <p>숫자 세기: {{ counter.count }}</p>
</template>
```


### 🌼 속성 바인딩 🌼


→ 이중 중괄호는 텍스트 삽입에만 사용됨 / 속성을 동적 값에 바인딩 하려면 `v-bind`  디렉티브를 사용


→ 디렉티브는 v- 접두사로 시작하는 특수한 속성으로 뷰 템플릿 문법의 일부 / 텍스트 삽입과 유사하게 디렉티브 값은 컴포넌트의 상태에 접근할 수 있는 Javascript 표현식


→ : 뒤의 부분은 디렉티브의 인자로 이 속성은 컴포넌트 상태의 = 뒤의 속성과 동기화됨


→ v-bind 는 줄여서 `:` 로 사용 가능


```javascript
<script>
export default {
  data() {
    return {
      titleClass: 'title'
    }
  }
}
</script>

<template>
  <h1 
:class
="titleClass">나를 빨갛게 만들어 보세요</h1>
</template>

<style>
.title {
  color: red;
}
</style>
```


→ data 안에는 return 값을 써줘야함


### 🌼 이벤트 리스너 🌼


→ `v-on` 디렉티브를 사용하여 DOM 이벤트를 수신할 수 있음


→ 단축키는  `@`


→ 메서드 내에서 `this` 를 사용하여 컴포넌트 인스턴스에 접근할 수 있음 / 컴포넌트 인스턴스는 data에 의해 선언된 데이터 속성을 노출 / 이러한 속성을 변경하여 컴포넌트 상태를 업데이트 할 수 있음


→ 이벤트 핸들러는 인라인 표현식을 사용할 수도 있으며 수식어를 사용하여 일반적인 작업을 단순화할 수 있음


```javascript
<script>
export default {
  data() {
    return {
      count: 0
    }
  },
  methods: {
    increment(){
      this.count ++
    }
  }
}
</script>

<template>
  <!-- 이 버튼이 작동하도록 만들어 봅시다 -->
  <button @click="increment">숫자 세기: {{ count }}</button>
</template>
```


→ methods : {} data 뒤에 , 찍고 씀 / increment(){}


### 🌼 폼 바인딩 🌼


→ `v-bind` + `v-on` = `v-model`


→ 이걸 사용하면 폼 안의 입력 엘리먼트에 양방향 바인딩을 만들 수 있음


→ `v-model` 은 입력 엘리먼트의 값을 바인딩된 상태와 자동으로 동기화하므로 더 이상 이에 대한 이벤트 핸들러를 사용할 필요가 없음


→ `v-model` 은 텍스트 입력 외에도 체크박스, 라디오 버튼, 셀렉트 드롭다운과 같은 다른 입력 타입에서도 작동


```javascript
<script>
export default {
  data() {
    return {
      text: ''
    }
  }
}
</script>

<template>
  <input v-model="text" placeholder="여기에 입력하기">
  <p>{{ text }}</p>
</template>
```


→ `v-bind` 뒤에 올 속성을 `v-model` 뒤에 적어주면 하나만 써도 됨


### 🌼 조건부 렌더링 🌼

```javascript
<script>
export default {
  data() {
    return {
      awesome: true
    }
  },
  methods: {
    toggle() {
      
this
.awesome = !
this
.awesome
    }
  }
}
</script>

<template>
  <button @click="toggle">토글 버튼</button>
  <h1 v-if="awesome">Vue는 굉장해! 엄청나!</h1>

</template>
```


→ `data` 안의 값을 `methods` 안으로 가져오고 싶을 때는 `this` 사용 기억하기


### 🌼 리스트 렌더링 🌼


→ v-for 디렉티브를 사용하여 자료 배열을 엘리먼트 목록으로 렌더링할 수 있음


```javascript
<ul>
	<li v-for="todo in todos" :key="todo.id">
		{{ todo.text }}
	</li>
</ul>
```


→ `todo` 는 **현재 반복 중인 엘리먼트**를 나타내는 로컬 변수 / 함수 범위와 유사하게 `v-for` 엘리먼트 위 또는 내부에서만 엑세스 가능 / 각 todo 객체에 고유한 `id` 를 부여하고 각 `li` 에 `key` 바인딩 / `key` 를 사용하면 Vue 가 각 `li` 를 정확하게 이동시켜 배열에서 해당 객체의 위치와 일치하도록 할 수 있음

> **목록을 업데이트 하는 두 가지 방법**
1. 자료 배열에서 변경 메서드 (mutating methods) 호출
`this.todos.push(newTodo)`
2. 배열을 새 배열로 교체
`this.todos = this.todos.filter()`

```javascript
<script>
// 각 할 일에 고유한 ID 부여
let id = 0

export default {
  data() {
    return {
      newTodo: '',
      todos: [
        { id: id++, text: 'HTML 배우기' },
        { id: id++, text: 'JavaScript 배우기' },
        { id: id++, text: 'Vue 배우기' }
      ]
    }
  },
  methods: {
    addTodo() {
      this.todos.push({ id: id++, text: this.newTodo }) //추가하고
      this.newTodo = '' //입력창 값을 비우기
    },
    removeTodo(todo) { //todo 는 todos 안의 객체 / 내가 클릭한 t 가 todo 랑 같지 않은 애들만 모아서 새로운 배열 / 그냥 내가 클릭한 애만 빼고 새로운 배열 만들어 -> 삭제기능이다 외우자}
      this.todos = this.todos.filter((t) => t !== todo)
    }
  }
}
</script>

<template>
  <form @submit.prevent="addTodo">
    <input v-model="newTodo">
    <button>할 일 추가</button>
  </form>
  <ul>
    <li v-for="todo in todos" :key="todo.id">
      {{ todo.text }}
      <button @click="removeTodo(todo)">X</button>
    </li>
  </ul>
</template>
```
