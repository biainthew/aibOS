### 서브넷팅


호스트 비트를 빌려서 서브넷을 만드는 것


### 기본 개념 정리


네트워크 : `192.168.10.0/24` 로 가정했을 때


`/24` :


```plain text
네트워크(24비트)              | 호스트(8비트)
11111111.11111111.11111111 | 00000000
고정된 부분                   | 변할 수 있는 부분
```


- 앞의 24비트는 네트워크 부분(고정)


- 뒤의 8비트는 호스트 부분


**만약** **`/26`** **이었다면 :**


```plain text
네트워크(24비트)              | 서브넷(2) | 호스트(6)
11111111.11111111.11111111 | 11      | 000000
```


- 서브넷 비트도 네트워크 부분에 포함됨


**2비트를 서브넷용으로 쓴다는 말은 무슨 말인지 :**


```plain text
[네트워크 24비트] [서브넷 2비트] [호스트 6비트]
                  ↑
                  이 2비트로 서브넷을 구분한다는 말
```


**예를 들어 :** 


```plain text
서브넷 1: 192.168.10.[00]000000 ~ [00]111111  (00은 고정)
서브넷 2: 192.168.10.[01]000000 ~ [01]111111  (01은 고정)
서브넷 3: 192.168.10.[10]000000 ~ [10]111111  (10은 고정)
서브넷 4: 192.168.10.[11]000000 ~ [11]111111  (11은 고정)
```


정리하자면 서브넷 비트는 어느 서브넷인지 구분하는 용도이고


각 서브넷 내에서는 **고정값**을 가짐 → 네트워크 부분으로 포함됨


### 예시


네트워크 : `192.168.10.0/24` 일 때 각 서브넷 당 **40대**를 수용해야 한다


**질문 1. 최소 몇 비트의 서브넷 마스크가 필요한가?**


    1-1. 만약 2비트를 서브넷용으로 쓴다면 ?


    ```plain text
    [2비트(서브넷) | 6비트(호스트)]
    ```


    - 호스트 비트 6개 → 2의 6승 = 64개


    - 사용가능 : 64-2 = 62개(네트워크 주소, 브로드캐스트 주소 제외)


    1-2. 만약 3비트를 서브넷용으로 쓴다면 ?


    ```plain text
    [3비트(서브넷) | 5비트(호스트)]
    ```


    - 호스트 비트 5개 → 2의 5승 = 32개


    - 사용가능 : 32-2 = 30개(네트워크 주소, 브로드캐스트 주소 제외)


    1-3. 결론


    최소 2비트를 서브넷용으로 사용해야 함


    기존 /24 + 서브넷 2비트 = /26


    **정답 : 26비트**


질문 2. 생성 가능한 서브넷의 개수는 최대 몇 개인가 ?


    정답 : 2의 2승, 4개


질문 3. 각 서브넷당 사용 가능한 호스트 수는 ?


    정답 : 2의 6승 - 2 = 62개
