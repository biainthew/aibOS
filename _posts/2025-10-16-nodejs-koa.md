## 🍏 백엔드 🍏

> 우리는 서버를 만들어 데이터를 여러 사람과 공유하지만 서버에 데이터를 담을 때는 여러 `규칙`이 필요함  
> 예를 들어 특정 `데이터를 등록`할 때 사용자 인증 정보가 필요할 수도 있고 등록할 데이터를 어떻게 검증할지 데이터의 종류가 다양하다면 어떻게 구분할지 등을 고려해야 함  
> `데이터를 조회`할 때도 어떤 종류의 데이터를 몇개씩 보여줄지 어떻게 보여줄지 등에 관할 것들을 고려해야함  
> 이런 것들에 관한 로직을 만드는 것을 `서버 프로그래밍` 또는 `백엔드 프로그래밍` 이라고 함  
> `백엔드 프로그래밍`은 언어에 구애받지 않아서 PHP 파이썬 Golang  자바 자바스크립트 루비 등과 같은 다양한 언어로 구현할 수 있음

## 🍏 Nod.js 🍏

> 처음에는 자바스크립트를 웹 브라우저에서만 사용했지만 계속 발전했고 구글이 크롬 웹 브라우저를 소개하면서 `V8` 이라는 자바스크립트 엔진도 공개  
> 이 자바스크립트 엔진을 기반으로 웹 브라우저 뿐만 아니라 서버에서도 자바스크립트를 사용할 수 있는 런타임을 개발했는데 그게 바로 `Node.js`

## 🍏 Koa 🍏

> `Node.js` 환경에서 웹 서버를 구축할 때는 보통 Express Hapi Koa 등의 웹 프레임워크를 사용함  
> `Koa` 는 `Express` 의 기존 개발팀이 개발한 프레임워크로 기존 Express 에서 고치고 싶었던 점들을 개선하면 내부 설계가 완전히 바뀌기 때문에 새로운 프레임워크를 개발했다고 함  
> `Express` 는 미들웨어 라우팅 템플릿 파일 호스팅 등과 같은 다양한 기능이 자체적으로 내장되어 있는 반면 `Koa` 는 미들웨어 기능만 갖추고 있으며 나머지는 다른 라이브러리를 적용하여 사용함 즉 우리가 필요한 기능들만 붙여서 서버를 만들 수 있기 때문에 Express 보다 훨씬 가벼움 그리고 `async await` 문법을 정식으로 지원하기 때문에 비동기 작업을 더 편하게 관리할 수 있음

### 🍏 Koa 기본 사용법 🍏


```javascript
const Koa = require('koa');
const app = new Koa();

app.use((ctx) => {
    ctx.body = 'hello world';
});

app.listen(4000, () => {
    console.log('Listening to port 4000');
});
```


🥬 서버를 포트 4000 번으로 열고 접속하면 hello world 를 반환하도록 설정
