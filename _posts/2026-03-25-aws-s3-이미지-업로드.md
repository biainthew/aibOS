게시글에 이미지를 첨부할 수 있도록 AWS S3를 연동한 과정 정리


### S3를 사용한 이유


이미지를 EC2에 직접 저장하면 두 가지 문제가 있음

- EC2 스토리지는 30GB라서 이미지가 많아지면 용량이 부족해짐
- EC2를 재생성하면 저장된 이미지가 모두 사라짐

S3는 용량 제한 없이 파일을 저장할 수 있고 EC2와 독립적으로 운영되기 때문에 이미지 저장소로 사용했음


### 전체 흐름


{% raw %}
```html
클라이언트 → POST /api/images (MultipartFile)
→ S3Service.upload()
→ S3 버킷에 파일 저장
→ S3 URL 반환
→ 클라이언트에서 URL을 게시글 본문에 포함
```
{% endraw %}


### S3 버킷 설정


| 항목         | 설정값                 |
| ---------- | ------------------- |
| 버킷 이름      | noticeboard-bucket  |
| 리전         | ap-northeast-2 (서울) |
| 퍼블릭 액세스 차단 | 해제                  |


퍼블릭 액세스를 해제하고 아래 버킷 정책을 적용해서 저장된 이미지를 누구나 URL로 접근할 수 있도록 설정함


{% raw %}
```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::noticeboard-bucket/*"
        }
    ]
}
```
{% endraw %}


### IAM 사용자 설정


Spring Boot에서 S3에 접근하기 위해 `AmazonS3FullAccess` 권한을 가진 IAM 사용자를 생성하고 액세스 키를 발급받음


액세스 키는 GitHub Secrets에 저장하고 GitHub Actions에서 환경변수로 주입하는 방식으로 관리함


| GitHub Secret  | 설명            |
| -------------- | ------------- |
| AWS_ACCESS_KEY | IAM 액세스 키     |
| AWS_SECRET_KEY | IAM 시크릿 액세스 키 |

### 주요 코드


**S3Config.java**


{% raw %}
```java
@Configuration
public class S3Config {

    @Value("${cloud.aws.credentials.access-key}")
    private String accessKey;

    @Value("${cloud.aws.credentials.secret-key}")
    private String secretKey;

    @Value("${cloud.aws.region.static}")
    private String region;

    @Bean
    public AmazonS3Client amazonS3Client() {
		    // 1. 자격 증명 객체 생성
        BasicAWSCredentials credentials = new BasicAWSCredentials(accessKey, secretKey);
        
        // 2. 빌더를 통해 클라이언트 구성 및 생성
        return (AmazonS3Client) AmazonS3ClientBuilder.standard()
                .withRegion(region) // 지역 설정
                .withCredentials(new AWSStaticCredentialsProvider(credentials)) // 인증 정보 설정
                .build(); // 최종 객체 생성
    }
}
```
{% endraw %}


Spring boot 어플리케이션에서 AWS S3 를 사용하기 위해 연결 설정을 하는 클래스 (AWS SDK V1 방식)

- `BasicAWSCredentials` : 입력받은 키들을 이용해 인증 객체 생성
- `AmazonS3ClientBuilder` : S3 클라이언트를 만들기 위해 지역과 인증 정보 세팅
- `withCredentials` : 생성한 인증 정보를 클라이언트에 주입해서 이 클라이언트를 사용할 때마다 AWS가 허가된 사용자임을 알 수 있게 함

**S3Service.java**


{% raw %}
```java
public String upload(MultipartFile file, String dirName) throws IOException {
    // UUID로 파일명 중복 방지
    String fileName = dirName + "/" + UUID.randomUUID() + "_" + file.getOriginalFilename();

    ObjectMetadata metadata = new ObjectMetadata();
    metadata.setContentType(file.getContentType());
    metadata.setContentLength(file.getSize());

    amazonS3Client.putObject(bucket, fileName, file.getInputStream(), metadata);

    return amazonS3Client.getUrl(bucket, fileName).toString();
}

public void delete(String fileUrl) {
    String fileName = fileUrl.substring(fileUrl.indexOf(bucket) + bucket.length() + 1);
    amazonS3Client.deleteObject(bucket, fileName);
}
```
{% endraw %}

- 파일명에 UUID를 붙이는 이유는 같은 이름의 파일을 업로드했을 때 기존 파일이 덮어씌워지는 것을 방지하기 위함
- `ObjectMetadata` 로 S3 가 파일을 제대로 인식할 수 있게 파일의 타입과 크기를 알려줌
- `putObject` 로 S3 에 실제로 업로드 후 해당 파일에 접근할 수 있는 URL 을 반환

### application.yml 설정


{% raw %}
```yaml
cloud:
  aws:
    s3:
      bucket: noticeboard-bucket
    region:
      static: ap-northeast-2
    stack:
      auto: false
    credentials:
      access-key: ${AWS_ACCESS_KEY}
      secret-key: ${AWS_SECRET_KEY}
```
{% endraw %}


`stack.auto: false` 로 설정하는 이유는 Spring Cloud AWS가 기본적으로 AWS CloudFormation 스택 정보를 자동으로 읽으려 하는데 이 기능이 필요 없는 경우 비활성화하지 않으면 애플리케이션 시작 시 에러가 발생하기 때문


### 추가로 보완하고 싶은 것들

1. 업로드 중 에러 발생할 경우 고려한 예외처리 추가
2. 현재는 s3 버킷 설정을 public 으로 해두었는데 보안을 고려한 설정 방식으로 변경
3. 파일 유효성 검사 (확장자, 크기 제한 등)
