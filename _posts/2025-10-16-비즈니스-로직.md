### 의미

1. 어플리케이션에서 비즈니스 규칙과 관련된 모든 로직을 처리하는 부분
2. 시스템이 수행해야 하는 핵심 작업과 규칙을 정의하는 코드

### 중요한 이유

1. 비즈니스 요구사항을 일관되게 적용할 수 있다
    1. ex) 고객 할인 정책, 환불 규정, 배송 방법 등
2. 비즈니스 로직은 시스템의 핵심 부분으로 여러 곳에서 재사용된다. 이를 하나의 독립된 로직으로 관리하면 코드 중복을 줄이고 유지보수가 쉬워진다
3. 유지보수가 용이해진다
4. 변경이 용이해진다
5. 어플리케이션의 가독성이 높아진다

### 예시


```java
public void transferFunds(Account fromAccount, Account toAccount, double amount) {
    if (fromAccount.getBalance() >= amount) {
        fromAccount.withdraw(amount);
        toAccount.deposit(amount);
    } else {
        throw new InsufficientFundsException("잔액이 부족합니다.");
    }
}
```


### 다른 로직과의 차이점

1. 비즈니스 로직 vs 데이터 접근 로직(DAO)
    1. 비즈니스 로직은 시스템이 무엇을 해야 하는지에 중점을 두는 반면 데이터 접근 로직은 데이터를 어떻게 가져오고 저장할 지에 초점을 둔다
2. 비즈니스 로직 vs UI 로직
    1. UI로직은 사용자의 상호작용과 관련된 부분을 처리하는 로직으로 UI 요소의 상태 변화나 사용자 입력에 대한 처리 등을 담당한다 ex) 버튼을 클릭하면 화면에 메세지를 띄운다
