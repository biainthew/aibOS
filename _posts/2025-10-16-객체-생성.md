OpinionItemDTO opinionItemDTO = new OpinionItemDTO();

OpinionItemDTO 라는 클래스의 인스턴스를 생성


자바에서 객체를 새로 만드는 작업


이 객체는 자식 데이터를 조회하기 위한 조건을 설정하거나 데이터를 담기 위해 사용된다


### 왜 쓰냐 ?

1. 필터 조건 설정

    데이터베이스에서 자식 데이터를 조회할 때 특정 조건을 기반으로 필터링 해야 하는 경우가 있음


    이 조건을 담는 용도로 OpinionItemDTO 객체를 생성하고 값을 설정함

2. 기존 메서드 구조 유지

    opinionRepository.selectOpinionItem 메서드가 OpinionItemDTO를 파라미터로 받도록 설계되었으므로, 새로운 OpinionItemDTO 객체를 만들어 그 구조를 따르는 것

3. 유연한 필터링

    여러 필터링 조건이 있을 수 있는데 이 객체를 생성하면 필요한 조건만 설정하고 메서드에 전달할 수 있음


    {% raw %}
```java
    opinionItemDTO.setOpinionInfoId(dto.getId());
    opinionItemDTO.setStatus("active"); // 예시: 추가 조건 설정
    ```
{% endraw %}


### 개선된 방식

1. 기존 방식

    {% raw %}
```java
    OpinionItemDTO opinionItemDTO = new OpinionItemDTO();
    opinionItemDTO.setOpinionInfoId(dto.getId());
    opinionItemDTO.setStatus("ACTIVE");
    List<OpinionItemDTO> items = opinionRepository.findItems(opinionItemDTO);
    ```
{% endraw %}

2. 빌더 사용

    {% raw %}
```java
    OpinionItemDTO opinionItemDTO = OpinionItemDTO.builder()
        .opinionInfoId(dto.getId())
        .status("ACTIVE")
        .build();
    List<OpinionItemDTO> items = opinionRepository.findItems(opinionItemDTO);
    ```
{% endraw %}
