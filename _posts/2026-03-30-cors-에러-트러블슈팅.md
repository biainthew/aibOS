프론트엔드에서 백엔드 API를 호출할 때 CORS 에러가 발생한 문제를 해결한 과정


### 문제 상황


React 로컬 개발 서버(`http://localhost:5173`)에서 Spring Boot(`http://localhost:8000`)로 로그인 API를 호출했을 때 아래 에러 발생


{% raw %}
```shell
Access to fetch at 'http://localhost:8000/api/auth/login' from origin
'http://localhost:5173' has been blocked by CORS policy:
Response to preflight request doesn't pass access control check:
No 'Access-Control-Allow-Origin' header is present on the requested resource.
```
{% endraw %}


### 원인 분석


**1단계 - WebMvcConfigurer CORS 설정 추가**


`WebMvcConfigurer`를 구현한 `CorsConfig`를 추가


{% raw %}
```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOrigins("http://localhost:5173")
                .allowedMethods("GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .allowCredentials(true)
                .maxAge(3600);
    }
}
```
{% endraw %}


그래도 에러가 계속 발생


**2단계 - Spring Security CORS 설정 추가**


Spring Security가 자체적으로 CORS를 처리하기 때문에 `SecurityConfig`에도 CORS 설정을 추가해야 했음


{% raw %}
```java
http
    .cors(Customizer.withDefaults())  // 추가
    ...
```
{% endraw %}


그래도 에러가 계속 발생


**3단계 - OPTIONS preflight 요청 허용**


브라우저는 실제 요청 전에 OPTIONS 메서드로 서버에 허용 여부를 먼저 확인하는 preflight 요청을 보냄 Spring Security가 이 OPTIONS 요청을 인증 필터로 막아버리기 때문에 `Access-Control-Allow-Origin` 헤더가 응답에 포함되지 않았던 것


### 해결


`SecurityConfig`의 `authorizeHttpRequests`에 OPTIONS 요청을 `permitAll()`로 열어주는 방식으로 해결함


{% raw %}
```java
.authorizeHttpRequests(auth -> auth
        .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()  // 추가
        .requestMatchers(
                "/api/auth/**",
                "/swagger-ui/**",
                "/swagger-ui.html",
                "/v3/api-docs/**"
        ).permitAll()
        .anyRequest().authenticated()
)
```
{% endraw %}


OPTIONS 요청은 실제 데이터를 주고받는 요청이 아니라 이 요청 허용해줄 수 있어? 라고 묻는 사전 확인 요청이기 때문에 인증 없이 열어두는 것이 보안상 문제없다고 함


### 정리


Spring Boot에서 CORS를 올바르게 설정하려면 세 가지가 모두 필요함

1. `WebMvcConfigurer`로 허용할 Origin, Method, Header 설정
2. `SecurityConfig`에 `.cors(Customizer.withDefaults())` 추가
3. OPTIONS 요청을 `permitAll()`로 허용
